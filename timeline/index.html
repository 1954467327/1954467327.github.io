1/21/2026: Linux互信
Category: LinuxTag: 互信

此文档介绍互信的原理，并提供批量互信脚本支持一键对多个服务器进行单向互信，服务器密码可以使用统一密码也可以使用自定义密码

1. 互信原理

1.1 SSH公钥认证机制

  • Linux互信基于SSH协议的公钥认证机制，其核心原理是：
  • 非对称加密：使用RSA或ED25519等算法生成公钥-私钥对
  • 身份验证：客户端用私钥签名，服务端用公钥验证签名
  • 免密登录：验证成功后无需输入密码即可建立SSH连接

1.2 工作流程

  1. 客户端发起SSH连接请求
  2. 服务端返回随机字符串(challenge)
  3. 客户端用私钥对字符串进行数字签名
  4. 服务端用存储的公钥验证签名
  5. 验证通过 → 建立连接；验证失败 → 要求密码认证

1/18/2026: Go语言的继承
Category: 后端Tag: Goland

当多个结构体存在相同的属性(字段)和⽅法时,可以从这些结构体中抽象出结构体,在该结构体中定义这些相同的属性和⽅法，其它的结构体不需要重新定义这些属性和⽅法，只需嵌套⼀个匿名结构体即可。也就是说:在Golang中，如果⼀个struct嵌套了另⼀个匿名结构体，那么这个结构体可以直接访问匿名结构体的字段和⽅法，从⽽实现了继承特性。1

继承语法说明

  1. 结构体可以使⽤嵌套匿名结构体所有的字段和⽅法，即:⾸字⺟⼤写或者⼩写的字段、⽅法，都可以使⽤。
  2. 匿名结构体字段访问可以简化。如：cat.Age --->cat对应的结构体中找是否有Age字段，如果有直接使⽤，如果没有就去找嵌⼊的结构体类型中的Age
1/18/2026: Go语言的封装
Category: 后端Tag: Goland

封装(encapsulation)就是把抽象出的字段和对字段的操作封装在⼀起，数据被保护在内部,程序的其它包只有通过被授权的操作⽅法，才能对字段进⾏操作。

封装的好处：

  1. 隐藏实现细节
  2. 提可以对数据进⾏验证，保证安全合理

Go语言如何实现封装

  1. 结构体和字段（属性）的首字母小写（小写代表其他包不可使用，本包的其他函数还是可以使用，封装没那么严格，类似于Java的private）
  2. 给结构体所在包提供⼀个⼯⼚模式的函数，⾸字⺟⼤写（类似⼀个构造函数）
  3. 提供一个首字母大写的Set方法（类似于Java的public），用于设置属性的值
  4. 提供⼀个⾸字⺟⼤写的Get⽅法(类似其它语⾔的public)，⽤于获取属性的值
1/18/2026: Go语言的多态
Category: 后端Tag: Goland

变量(实例)具有多种形态。⾯向对象的第三⼤特征，在Go语⾔，多态特征是通过接⼝实现的。可以按照统⼀的接⼝来调⽤不同的实现。这时接⼝变量就呈现不同的形态。

接口定义

定义规则、定义规范，定义某种能⼒

接口说明

  1. 接口定义本身不能包含成员变量/字段，只能包含方法签名。但这不意味着方法签名中不能有参数。
// ✅ 正确的接口定义
type Writer interface {
	// 方法可以有参数（data）和返回值（n, err）
	Write(data []byte) (n int, err error)
}

// ❌ 错误的接口定义（编译不通过）
type BadInterface interface {
	buffer []byte        // 错误！接口不能包含字段
	counter int          // 错误！接口不能包含字段
}
1/16/2026: 错误码规范
Category: 后端Tag: 开发规范, 自定义错误码

由于业务系统错误码定义不一，也无法区别业务错误和技术错误

格式定义

错误码长度为8位，组成格式为1+2+2+3的方式。

  1. 第一位长度为1代表错误码为业务类还是技术类，用E表示业务，S表示技术。
  2. 第二位长度为2，代表系统标识，此处本系统暂填写固定值LT
  3. 第三位长度为2，代表错误码的小类，分为业务类和技术类，具体参考附录中的错误码三级分类表。
  4. 第四位长度为3，代表具体小类编号，企架规范可为数字或字母，本系统使用001~999来定义，如超出再使用字母+数字编号。
